# templates

開発ベースライン設定を複数リポジトリへ配布するためのテンプレートリポジトリです。

`templates.yaml` を source of truth とし、Go 製 CLI `template-sync` で GitHub Actions、commitlint、lefthook、pinact、mise、Entire などの設定ファイルを同期します。

## Included templates

現在の同期対象は `templates.yaml` で管理しています。

| ID | Target | Notes |
| --- | --- | --- |
| `github-workflow-scan` | `.github/workflows/scan.yaml` | Betterleaks scan workflow |
| `github-workflow-semantic-pr` | `.github/workflows/semantic-pr.yaml` | Semantic PR title workflow |
| `commitlint` | `.commitlintrc.yaml` | Commit message rules |
| `lefthook` | `lefthook.yaml` | Git hooks for scan and commitlint |
| `pinact` | `.pinact.yaml` | Pinning rules |
| `mise-baseline` | `.mise/config.baseline.toml` | Shared mise tasks/env |
| `mise-project` | `.mise/config.toml` | Created only when missing |
| `mise-rc` | `.miserc.toml` | mise activation config |
| `LICENSE` | `LICENSE` | Shared license file |
| `entire-gitignore` | `.entire/.gitignore` | Entire ignore rules |
| `entire-settings-json` | `.entire/settings.json` | Entire shared settings |

`template-sync update` は、manifest の `gitignore` にある項目も配布先の `.gitignore` へ追加します。

## Requirements

- Go `1.22` 以降
- `git`（`template-sync diff` が `git diff --no-index` を使います）
- 任意: mise（このリポジトリでは Go `1.22.12` を pin しています）

## Quick start

このリポジトリで CLI を確認します。

```sh
go run ./src/cmd/template-sync -- help
go run ./src/cmd/template-sync -- check
```

配布先リポジトリでは、このテンプレートリポジトリを指定します。

```sh
go run ../templates/src/cmd/template-sync -- check \
  --template-dir ../templates \
  --target-dir .

go run ../templates/src/cmd/template-sync -- update \
  --template-dir ../templates \
  --target-dir . \
  --repository y-writings/templates \
  --ref v2026.05.01
```

ローカルバイナリを使う場合:

```sh
go build -o bin/template-sync ./src/cmd/template-sync
bin/template-sync help
```

## Commands

```text
template-sync check   # target が template と一致するか確認
template-sync diff    # 追加・更新予定の差分を表示
template-sync update  # ファイルを同期し、lock file を更新
template-sync prune   # manifest から消えた未変更ファイルを削除
```

主なオプションは `--template-dir`、`--target-dir`、`--manifest`、`--lock`、`--repository`、`--ref` です。

## How it works

`template-sync` はファイル単位の同期ツールです。テンプレート変数の展開や YAML/TOML の部分マージはせず、source file を byte-for-byte でコピーします。

同期状態は配布先の `.template-sync.lock` に記録されます。

```yaml
repository: y-writings/templates
ref: v2026.05.01
files:
  github-workflow-scan:
    target: .github/workflows/scan.yaml
    source_sha256: ...
```

主な status は `synced`、`add`、`update`、`prune`、`conflict` です。`update` は stale file を削除しないため、manifest から消えたファイルの削除には `prune` を明示的に使います。

安全のため、manifest の source/target と lock target は各 root 配下の相対パスだけを許可します。

詳しい設計背景は [`docs/template-sync.md`](docs/template-sync.md) を参照してください。現在の実装と `templates.yaml` が古い設計例と異なる場合は、実装と `templates.yaml` を正とします。

## Development

```sh
mise install
mise run baseline:setup

go test ./...
bash scripts/test-derive-release-tag.sh

go run ./src/cmd/template-sync -- check
go run ./src/cmd/template-sync -- diff
go run ./src/cmd/template-sync -- update --repository y-writings/templates --ref v2026.05.01
mise run baseline:pin
mise run baseline:scan
```

## Maintainer operations

- Snapshot tags use `vYYYY.MM.DD` and are driven by `.github/workflows/release-tag.yml`.
- `CHANGELOG.md` is generated by `git-cliff` through `.github/workflows/changelog.yml`.
- Repository setup details are in [`docs/github-settings.md`](docs/github-settings.md).

Snapshot helper:

```sh
SNAPSHOT_DATE=2026.05.01 bash scripts/derive-release-tag.sh
```
